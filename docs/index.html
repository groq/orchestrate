Quick Start

1. Create settings.orchestrate.yaml in the Orchestrate data directory:

macOS: ~/Library/Application Support/Orchestrate/
Linux: ~/.local/share/orchestrate/
Windows: %APPDATA%\Orchestrate\ 

default: default

presets:
  # Simple: just an agent
  default:
    - agent: claude

  # Complex: agent with dev server, tests, and shell
  fullstack:
    - agent: claude
      commands:
        - command: "npm run dev"
          title: "Dev Server"
        - command: "npm run test:watch"
          title: "Tests"
        - command: ""
          title: "Shell"

  # Multi-agent: compare different agents on the same task
  compare:
    - agent: droid
    - agent: claude
    - agent: codex

  # Best of n: use n to run multiple instances of an agent
  scale:
    - agent: claude
      n: 3

2. Run with a GitHub repo:

# Use default preset
orchestrate --repo groq/openbench --name fix-bug --prompt "Fix the login timeout issue"

# Or use the fullstack preset
orchestrate --repo groq/openbench --name fix-bug --prompt "Fix the login timeout issue" --preset fullstack

This clones/updates the repo from the main branch, creates isolated git worktrees, and launches agents/commands in separate iTerm2 panes.

Installation

go install github.com/groq/orchestrate@latest

Requirements: macOS with iTerm2, Go 1.21+, and your preferred AI coding agents installed.

CLI Reference

Flag Description
--repo Required. GitHub repo to clone (e.g., groq/openbench). Clones fresh or updates from main branch.
--name Required. Branch name prefix for worktrees. Each branch gets a unique hex suffix.
--prompt Required. The prompt to pass to each agent.
--preset Use a preset from settings.orchestrate.yaml. Defaults to the config's default preset.
--n Multiplier for agent windows. --n 2 runs each agent twice.

Examples

Basic Usage

# Use default preset
orchestrate --repo myorg/myapp --name feature-auth --prompt "Add OAuth2 login"

Using Presets

# Use the "dev" preset with commands
orchestrate --repo myorg/myapp --name big-task --preset dev --prompt "Implement full test coverage"

Scale with Multiplier

# Run each agent in the preset 3 times
orchestrate --repo myorg/myapp --name massive --preset mix --n 3 --prompt "Optimize queries"
Each agent runs in its own git branch, so they won't conflict. Compare results and pick the best solution!

Configuration

Create settings.orchestrate.yaml in the Orchestrate data directory to define presets. This file is required.

# .orchestrate.yaml

# Default preset when --preset is not specified
default: standard

# Presets define agents and their associated commands
presets:
  # Simple: just agents
  standard:
    - agent: droid
    - agent: codex
    - agent: codex

  # With commands: agents + terminals in their worktrees
  dev:
    - agent: codex
      commands:
        - command: "./bin/go run ./cmd/myapp"
          title: "App"
        - command: ""
          title: "Terminal"

Agents with Commands

Each agent can have commands that run in its worktree. Commands are nested under the agent:

presets:
  dev:
    - agent: claude
      commands:
        # These run in claude's worktree
        - command: "npm run dev"
          title: "Dev Server"
          color: "#00ff00"
        - command: "npm run test:watch"
          title: "Tests"
          color: "#ffff00"
    
    - agent: codex
      commands:
        # Opens an empty terminal in codex's worktree
        - command: ""
          title: "Terminal"

Agent Options

agent Agent name (droid, claude, codex)
n Multiplier: run this agent N times (default: 1)
commands List of commands to run in this agent's worktree

Command Options

command Shell command to run (empty string = just open terminal)
title Custom title for the terminal tab
color Hex color for the tab, e.g., #ff8800
Commands always run in their parent agent's worktree and show the branch name in the title.

How It Works

1. Clone & Update

Orchestrate clones the specified GitHub repo (or updates if it already exists), always fetching the latest from the main branch.

2. Git Worktrees

Creates isolated git worktrees, each with a unique branch based on main. Worktrees are stored in a platform-appropriate location.

3. iTerm2 Integration

Windows open automatically with up to 6 panes each in a grid layout. Each pane is color-coded by agent type or custom color.

4. Parallel Execution

Agents and commands run in the exact order defined. When agents finish, compare branches and merge the best solution.

Data Location

Orchestrate stores all data in platform-standard locations:

  • macOS: ~/Library/Application Support/Orchestrate/
  • Linux: ~/.local/share/orchestrate/
  • Windows: %APPDATA%\Orchestrate

Inside this directory:

  • settings.orchestrate.yaml ‚Äî Required. Your presets configuration
  • repos/ ‚Äî Cloned repositories
  • worktrees/ ‚Äî Git worktrees for agent sessions
